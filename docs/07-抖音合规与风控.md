## 一、合规背景与政策环境

### 1.1 核心法规梳理

| 法规/政策 | 生效时间 | 核心要求 | 对本项目影响 |
|-----------|----------|----------|-------------|
| 《直播电商监督管理办法》 | 2026-02-01 | AI生成内容必须标识，数字人主播纳入监管 | **直接影响**：必须在直播画面标识AI内容 |
| 《互联网信息服务深度合成管理规定》 | 2023-01-10 | 深度合成内容需标识，提供者需备案 | 需对AI生成的语音、图像进行标识 |
| 《生成式人工智能服务管理暂行办法》 | 2023-08-15 | 生成式AI服务需备案，内容需合规 | AI解说、AI决策内容需合规审查 |
| 抖音《虚拟主播管理细则》 | 2025年更新 | 虚拟主播需备案，真人运营者需实名 | 需完成虚拟主播备案流程 |
| 抖音《直播行为规范》 | 持续更新 | 禁止赌博、低俗、诱导消费等 | 棋牌内容需严格规避赌博嫌疑 |

### 1.2 2025-2026年监管趋势

根据《2025抖音直播治理白皮书》数据：

| 指标 | 数据 | 启示 |
|------|------|------|
| 全年封禁违规账号 | 37万个 | 平台治理力度极大 |
| AI审核准确率 | >90% | 违规内容极难逃过审核 |
| 违规曝光量下降 | 37% | 生态持续净化 |
| 健康分覆盖提升 | 60% | 健康分机制是核心管理手段 |

| 协助警方捣毁黑产 | 39个团伙 | 涉赌涉诈零容忍 |

### 1.3 本项目合规定位

```
核心定位: AI科技娱乐直播（非棋牌赌博直播）
关键区分:
  ✅ AI技术展示 + 娱乐互动 = 合规
  ❌ 棋牌赌博 + 真金白银 = 违规

我们是: "看AI打牌的科技娱乐节目"
我们不是: "线上棋牌赌博直播间"
```

## 二、AI 内容标识规范

### 2.1 标识要求（强制）

根据《直播电商监督管理办法》第X条，使用AI生成内容必须进行标识。

| 标识项 | 标识方式 | 位置 | 持续时间 |
|--------|----------|------|----------|
| AI内容总标识 | 画面左上角固定水印 "AI生成内容" | 左上角 | 全程 |
| AI角色标识 | 每个AI角色名称旁标注"AI" | 角色头像旁 | 全程 |
| AI解说标识 | TTS语音播放时显示"AI解说" | 解说面板 | 解说播放时 |
| AI决策标识 | 出牌时标注"AI决策" | 出牌区域 | 每次出牌 |

### 2.2 标识设计规范

```
视觉要求:
- 字体大小: ≥ 画面高度的 3%
- 对比度: 与背景色对比度 ≥ 4.5:1
- 透明度: 不低于 70%
- 位置: 不被其他元素遮挡
- 字体: 清晰可辨的无衬线字体

标识文案（推荐）:
- 主标识: "本直播间内容由AI生成"
- 角色标识: "[AI] 烈焰哥" / "[AI] 冰山姐" / "[AI] 戏精弟"
- 解说标识: "🤖 AI解说中"
- 开场提示: "本直播间所有角色均为AI虚拟角色，非真人"
```

### 2.3 开场合规声明（每场必播）

```
文案模板:
"欢迎来到AI大乱斗直播间！温馨提示：本直播间所有参赛选手均为
AI虚拟角色，所有对局内容由AI自动生成。本直播不涉及任何真实
赌博行为，游戏积分为虚拟积分，无任何现金价值。祝大家观看愉快！"

播放要求:
- 每场直播开始时播放一次（TTS语音 + 文字弹窗）
- 每2小时重复播放一次简短版
- 简短版: "温馨提示：本直播间内容由AI生成，不涉及真实赌博。"
```

## 三、虚拟主播备案

### 3.1 备案流程

```
步骤1: 准备材料
  - 运营者身份证正反面
  - 营业执照（如有）
  - 虚拟形象设计稿（3个AI角色）
  - AI技术说明文档
  - 内容安全承诺书

步骤2: 抖音创作者服务中心提交备案
  - 路径: 抖音APP → 创作者服务中心 → 虚拟主播备案
  - 填写虚拟形象信息、技术方案、运营计划

步骤3: 平台审核（预计3-7个工作日）
  - 形象审核: 不涉及色情、暴力、政治敏感
  - 技术审核: AI内容生成方式合规
  - 运营审核: 直播内容计划合规

步骤4: 获得备案编号
  - 备案编号需在直播间展示
  - 备案有效期通常为1年，需定期续期
```

### 3.2 备案材料清单

| 材料 | 要求 | 说明 |
|------|------|------|
| 运营者实名信息 | 身份证 + 手机号 | 必须与抖音账号实名一致 |
| 虚拟形象设计图 | 每个角色正面/侧面图 | 3个AI角色各一套 |
| 形象版权证明 | 原创声明或授权书 | 证明形象不侵权 |
| 技术方案说明 | AI决策、TTS、画面生成方式 | 说明非深度伪造真人 |
| 内容安全方案 | 内容审核机制说明 | 如何防止违规内容 |
| 运营计划书 | 直播时段、内容类型 | 说明是AI娱乐内容 |

### 3.3 备案注意事项

- 虚拟形象不得模仿真实公众人物
- 虚拟形象不得含有色情、暴力元素
- 备案信息变更需及时更新
- 未备案直接开播可能导致限流或封号

## 四、棋牌内容合规红线

### 4.1 绝对禁止行为（触犯即封号）

| 编号 | 禁止行为 | 风险等级 | 处罚 |
|------|----------|----------|------|
| R01 | 涉及真金白银赌博 | 🔴 致命 | 永久封号 + 报警 |
| R02 | 积分可兑换现金或实物 | 🔴 致命 | 永久封号 |
| R03 | 引导观众到外部赌博平台 | 🔴 致命 | 永久封号 + 报警 |
| R04 | 使用"赌""押注""下注"等赌博术语 | 🟠 高危 | 限流/封禁 |
| R05 | 观众礼物直接决定胜负结果 | 🟠 高危 | 限流/封禁 |
| R06 | 暗示观众可通过刷礼物"赢钱" | 🟠 高危 | 限流/封禁 |
| R07 | 展示赌博相关广告或二维码 | 🔴 致命 | 永久封号 |

### 4.2 高风险行为与规避策略

| 风险行为 | 为什么有风险 | 规避策略 |
|----------|-------------|----------|
| 使用"积分"一词 | 可能被理解为可兑换的筹码 | 改用"战绩分"或"荣誉值" |
| 观众"押注"某AI赢 | 类似赌博下注 | 改为"预测"，且不涉及任何价值交换 |
| 礼物影响比赛走向 | 可能被认为是"花钱买结果" | 礼物效果限于娱乐特效，不直接决定胜负 |
| 展示"倍数"概念 | 类似赌博赔率 | 仅作为游戏机制展示，强调"虚拟积分" |
| "翻倍""加倍"用语 | 赌博联想 | 改用"能量加成""战力提升" |
| 排行榜展示积分 | 类似赌场筹码排行 | 标注"虚拟荣誉积分，无任何现金价值" |

### 4.3 敏感词库（直播间禁用词）

| 类别 | 禁用词 | 替代词 |
|------|--------|--------|
| 赌博类 | 赌、赌博、赌局、赌注 | 对战、比赛、对局 |
| 下注类 | 押注、下注、投注、买 | 预测、支持、加油 |
| 金钱类 | 赢钱、输钱、赔率、筹码 | 赢分、失分、倍率、战绩 |
| 翻倍类 | 翻倍、加倍、双倍 | 能量加成、战力提升 |
| 诱导类 | 稳赚、包赢、必胜 | （禁止使用，无替代） |
| 兑换类 | 提现、兑换、变现 | （禁止使用，无替代） |

### 4.4 礼物互动合规边界

```
✅ 合规的礼物效果:
- 触发视觉特效（烟花、闪电等）
- 给AI角色加油（显示加油动画）
- 触发AI角色特殊表情/台词
- 影响游戏节奏（如触发限时出牌模式）
- 解锁观众投票权（投票选地主）

❌ 违规的礼物效果:
- 直接让某个AI获胜
- 让某个AI获得必胜手牌
- 礼物金额与游戏积分挂钩
- 礼物排行与游戏奖励挂钩
- 任何形式的"花钱买结果"

⚠️ 灰色地带（需谨慎处理）:
- 礼物触发"偷看牌" → 可以，但需标注"娱乐效果"
- 礼物触发"换牌" → 风险较高，建议仅换非关键牌
- 礼物触发"加倍" → 改为"能量加成"，纯视觉效果
```

## 五、主播健康分管理

### 5.1 健康分机制概述

抖音"主播健康分"是平台对主播行为的核心评价体系，直接影响直播间流量和权限。

| 分值区间 | 状态 | 影响 |
|----------|------|------|
| 100分 | 优秀 | 正常推荐流量，可申请优质认证 |
| 80-99分 | 良好 | 正常推荐流量 |
| 60-79分 | 警告 | 推荐流量下降，收到警告通知 |
| 40-59分 | 限制 | 严重限流，部分功能受限 |
| <40分 | 危险 | 可能被暂停直播权限 |
| 0分 | 封禁 | 直播权限被回收 |

### 5.2 扣分风险点与防范

| 扣分行为 | 扣分幅度 | 本项目风险 | 防范措施 |
|----------|----------|-----------|----------|
| 涉赌内容 | -20~-100 | 🟠 中高 | 严格执行敏感词过滤 |
| 低俗内容 | -10~-50 | 🟢 低 | AI角色言行预审 |
| 诱导消费 | -10~-30 | 🟡 中 | 礼物效果合规设计 |
| 未标识AI内容 | -5~-20 | 🟢 低 | 全程AI标识水印 |
| 违规互动 | -5~-20 | 🟡 中 | 弹幕过滤 + 互动规则限制 |
| 未成年人保护缺失 | -20~-50 | 🟢 低 | 开启青少年模式提示 |

### 5.3 健康分维护策略

```
日常维护:
1. 每日开播前检查健康分状态
2. 每周复盘扣分记录，分析原因
3. 保持稳定开播频率（平台偏好活跃账号）
4. 积极响应平台通知和整改要求

加分行为:
- 持续合规直播（连续30天无违规可恢复部分扣分）
- 申请优质/专业认证
- 参与平台官方活动
- 内容质量评分高（互动率、观看时长等）
```

## 六、内容安全审核机制

### 6.1 三层审核体系

```
┌─────────────────────────────────────┐
│  第一层: 预审（开播前）              │
│  - AI角色台词库预审                  │
│  - 解说文案模板审核                  │
│  - 敏感词库更新检查                  │
├─────────────────────────────────────┤
│  第二层: 实时审核（直播中）           │
│  - AI输出内容实时过滤                │
│  - 弹幕敏感词自动屏蔽               │
│  - 异常行为监控告警                  │
├─────────────────────────────────────┤
│  第三层: 复审（直播后）              │
│  - 直播录像抽检回看                  │
│  - 观众举报处理                      │
│  - 平台通知响应                      │
└─────────────────────────────────────┘
```

### 6.2 AI 输出内容过滤规则

| 过滤对象 | 过滤规则 | 处理方式 |
|----------|----------|----------|
| AI角色台词 | 敏感词库匹配 | 替换为安全词或跳过 |
| AI解说文案 | 模板化 + 变量填充 | 变量值白名单校验 |
| AI情绪表达 | 情绪枚举限制 | 仅允许预定义的8种情绪 |
| 弹幕内容 | 平台自带 + 自定义过滤 | 自动屏蔽 |
| 礼物触发文案 | 预设模板 | 不允许自定义文案 |

### 6.3 异常行为监控

| 监控项 | 阈值 | 告警方式 | 处理 |
|--------|------|----------|------|
| AI输出敏感词 | 单场≥3次 | 即时告警 | 暂停AI发言，人工介入 |
| 弹幕涉赌关键词 | 单分钟≥5条 | 即时告警 | 开启弹幕严格过滤 |
| 礼物异常集中 | 单用户5分钟内≥500元 | 延迟告警 | 触发冷静期提示 |
| 观众举报 | 单场≥10次 | 即时告警 | 人工审查直播内容 |
| 平台警告通知 | 任意一次 | 最高优先级 | 立即暂停直播，排查问题 |

## 七、风险应急预案

### 7.1 风险等级定义

| 等级 | 定义 | 响应时间 | 处理人 |
|------|------|----------|--------|
| P0 紧急 | 收到平台封号/永久限流通知 | 立即 | 项目负责人 |
| P1 严重 | 收到平台警告/临时限流 | 30分钟内 | 运营负责人 |
| P2 重要 | 健康分大幅下降/观众大量举报 | 2小时内 | 内容审核员 |
| P3 一般 | 单次敏感词触发/个别弹幕违规 | 当日处理 | 值班运营 |

### 7.2 P0 紧急预案：封号/永久限流

```
立即行动:
1. 停止所有直播推流
2. 截图保存平台通知内容
3. 导出近7天直播录像备查
4. 分析封号原因（查看平台给出的违规说明）

申诉流程:
1. 24小时内通过抖音创作者服务中心提交申诉
2. 准备申诉材料:
   - AI内容标识截图（证明已合规标识）
   - 虚拟主播备案编号
   - 内容安全方案文档
   - 直播录像片段（证明无违规内容）
3. 跟进申诉进度，必要时电话联系平台客服

备用方案:
- 启用备用账号（需提前注册并完成备案）
- 切换到其他平台（B站、快手）临时直播
- 通过短视频发布"停播通知"告知粉丝
```

### 7.3 P1 严重预案：平台警告/临时限流

```
立即行动:
1. 不要立即停播（避免影响健康分的开播稳定性指标）
2. 截图保存警告通知内容
3. 立即切换到"安全模式"直播:
   - 关闭所有礼物互动效果
   - AI解说切换为纯模板化文案（禁用任何动态生成内容）
   - 弹幕过滤等级调至最高
4. 当场直播结束后立即排查

排查清单:
□ 检查近3场直播的AI输出日志，搜索敏感词命中记录
□ 检查礼物互动是否触发了违规边界效果
□ 检查弹幕中是否有大量涉赌内容未被过滤
□ 检查AI标识水印是否全程正常显示
□ 检查是否有观众举报记录

整改措施:
- 根据排查结果更新敏感词库
- 调整AI输出过滤规则
- 必要时暂停特定互动功能
- 整改完成后恢复正常直播模式
```

### 7.4 P2 重要预案：健康分下降/大量举报

```
触发条件:
- 健康分单日下降 ≥ 10分
- 单场举报数 ≥ 10次
- 连续3场出现敏感词告警

处理流程:
1. 导出近期直播数据进行全面分析
2. 对比健康分变化与直播内容的关联
3. 针对性调整:
   - 如因"涉赌"扣分 → 加强敏感词过滤，调整用语
   - 如因"诱导消费"扣分 → 调整礼物互动文案
   - 如因"低俗内容"扣分 → 审查AI角色台词库
4. 调整后观察3天，确认健康分企稳
5. 如持续下降，升级为P1处理
```

### 7.5 P3 一般预案：单次违规/个别问题

```
处理流程:
1. 记录违规详情（时间、内容、触发原因）
2. 当日内修复:
   - 敏感词触发 → 加入词库或调整AI输出逻辑
   - 弹幕违规 → 加入弹幕过滤词库
   - 标识缺失 → 检查水印渲染逻辑
3. 更新运维日志
4. 纳入周度复盘

注意: P3问题虽然单次影响小，但累积效应不可忽视。
连续出现3次以上同类P3问题，应升级为P2处理。
```

## 九、合规技术实现要点

### 9.1 敏感词过滤系统架构

```
┌──────────────────────────────────────┐
│           敏感词过滤引擎              │
├──────────────────────────────────────┤
│  输入层:                              │
│  - AI角色台词 → 出牌前过滤           │
│  - AI解说文案 → TTS合成前过滤        │
│  - 弹幕内容 → 显示前过滤             │
├──────────────────────────────────────┤
│  过滤层:                              │
│  - 精确匹配: 完整词命中              │
│  - 模糊匹配: 谐音、变体、拼音        │
│  - 语义匹配: 上下文语义判断          │
├──────────────────────────────────────┤
│  处理层:                              │
│  - 替换: 用安全词替代敏感词           │
│  - 屏蔽: 直接不显示/不播放           │
│  - 告警: 记录日志 + 触发监控告警     │
└──────────────────────────────────────┘
```

### 9.2 AI标识水印技术要求

| 技术项 | 要求 | 实现方式 |
|--------|------|----------|
| 水印渲染 | 不可被OBS滤镜去除 | 前端Canvas直接渲染，非叠加层 |
| 水印位置 | 左上角固定，不随场景切换消失 | 全局UI层，z-index最高 |
| 水印内容 | "本直播间内容由AI生成" | 静态文本 + 半透明背景 |
| 抗干扰 | 不被弹幕、特效遮挡 | 独立渲染层，最高优先级 |
| 可配置 | 支持文案、位置、样式热更新 | 配置文件驱动，无需重启 |

> **渲染层级规范**：AI 标识水印必须渲染在前端分层架构的最高层——即 `02-前端界面设计 §2.1` 定义的 **Layer 6 - 系统提示层（z-index: 60）**。该层级高于礼物特效层（z:50）、特效层（z:40）、弹幕层（z:30）等所有业务层，确保水印在任何场景下均不被遮挡。技术实现上，水印应直接在 PixiJS Canvas 的最顶层容器中渲染，而非通过 CSS overlay 叠加，以防止被 OBS 滤镜或浏览器插件移除。

### 9.3 合规日志记录

```
必须记录的日志:
1. AI输出日志: 每条AI生成的文本内容 + 时间戳
2. 敏感词命中日志: 命中词条、原文、处理方式
3. 弹幕过滤日志: 被过滤的弹幕内容 + 过滤原因
4. 礼物事件日志: 礼物类型、触发效果、执行结果
5. 异常告警日志: 告警类型、触发条件、处理结果

日志保留期限: ≥ 90天（满足平台可能的回溯审查需求）
日志格式: JSON结构化，便于检索和分析
```

## 十、总结与核心原则

### 10.1 合规三原则

```
原则一: 主动合规，而非被动应对
  → 提前备案、主动标识、预防性过滤

原则二: 技术保障，而非人工兜底
  → 敏感词自动过滤、AI输出预审、异常自动告警

原则三: 持续迭代，而非一劳永逸
  → 每周更新词库、每月审计内容、跟踪政策变化
```

### 10.2 核心定位再强调

```
我们是: "AI科技娱乐直播" —— 看AI打牌的科技秀
我们不是: "线上棋牌赌博" —— 涉及真金白银的赌局

所有设计决策都应围绕这个定位展开:
- 用语: "对战"而非"赌局"，"战绩分"而非"筹码"
- 互动: "加油助威"而非"押注下注"
- 礼物: "娱乐特效"而非"影响胜负"
- 氛围: "科技感+趣味性"而非"刺激感+赌博感"
```
